## 해결된 주요 이슈

### 1. AI 지침 깨짐 문제 (2026-03-19 해결)

**증상**: 수정 요청 시 이전에 먹인 지침(인트로 제거, 7문단 구조, 산문체 등)이 깨지는 현상 빈도 증가

**원인 분석**:

- `process/route.ts`(초기 생성)와 `chat/route.ts`(수정 생성)의 시스템 프롬프트 불일치
- 초기 생성: temperature 0 / 수정: temperature 0.8 → 구조 자의적 변경 가능
- 초기 생성: max_tokens 8096 / 수정: 4096 → 출력 잘림 가능

**해결**: chat/route.ts의 할루시네이션 방지 규칙을 process/route.ts와 동일하게 강화, temperature 0으로 통일, max_tokens 8096으로 통일

### 2. 미리보기 콜아웃 문제 (2026-03-19 해결)

**증상**: 미리보기에서 blockquote(>) 사용 시 파란색 콜아웃 박스로 표시되어 가독성 저하

**해결**: blockquote 렌더링을 콜아웃 스타일에서 일반 텍스트로 변경하여 굵은 제목+본문 형태로 통일

## 현재 아키텍처

```
hospital-blog-automation/
├── src/
│   ├── app/
│   │   └── api/
│   │       ├── process/route.ts    # 초기 블로그 생성 (Claude Sonnet 4.5)
│   │       ├── chat/route.ts       # 수정 요청 처리 (Claude Opus 4.5)
│   │       └── requests/route.ts   # 요청 목록 CRUD
│   ├── components/
│   │   ├── admin/                  # 관리자 UI (직원 통계 등)
│   │   ├── dashboard/              # 대시보드
│   │   ├── requests/               # 요청 관리 (채팅 모달, 새 요청)
│   │   └── ui/                     # 공통 UI (마크다운 뷰어 등)
│   ├── lib/
│   │   ├── store.tsx               # 전역 상태 관리
│   │   ├── slack.ts                # Slack 알림
│   │   ├── google-drive.ts         # Google Drive 연동
│   │   └── mock-data.ts            # 개발용 목데이터
│   └── types/                      # TypeScript 타입 정의
```

**데이터 흐름**: Google Sheets(요청 저장) → Claude API(글 생성) → Google Drive(결과 저장) → Slack(알림)
